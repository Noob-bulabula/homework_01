# GDPVAL 专业任务执行增强 Skill

## 概述

GDPVAL 是专为知识密集型专业任务设计的执行增强 Skill，旨在解决大语言模型在复杂真实场景下常见的系统性缺陷（如幻觉、能力受限时强行完成、忽略 Prompt 约束、数据无来源等）。

核心目标：**在严格遵守 Prompt 要求的前提下，实现可靠、可追溯、可审计的任务执行**。

该 Skill 提供六大层级增强能力：

1. **需求层**：Prompt 严格解析与执行（最高优先级）
2. **感知层**：外部数据获取与参考文件处理
3. **推理层**：数据幻觉预防与约束理解
4. **行动层**：优雅降级与透明报告
5. **验证层**：输出可追溯性与事实核查
6. **集成层**：与其他文件生成 Skill（docx、xlsx、pdf 等）无缝协同

适用于金融分析、审计抽样、云架构设计、定量研究报告、策略规划等需要高准确性与专业性的任务。

## 特性亮点

- **Prompt 约束至上**：任何降级策略都不允许违反 Prompt 明确要求
- **零幻觉承诺**：宁可承认“无法获取”，绝不捏造数据或文件
- **能力自检**：任务开始前自动检测网络、文件、工具可用性
- **优雅降级**：能力受限时提供清晰影响评估与替代方案
- **评分系统兼容**：结构化输出 claude_output.md，确保可被自动化评估
- **多格式交付**：支持与 docx、xlsx、pdf 等 Skill 协同生成真实文件
- **全链路可追溯**：每个数据点强制标注来源，输出含完整血缘报告

## 核心执行流程

1. **需求解析** → 提取交付物、格式、参考文件、约束
2. **能力检测** → 网络、参考文件、生成工具可用性检查
3. **数据获取** → 优先工具调用 → 代理 fallback → 手动指导
4. **内容生成** → 严格参照 Prompt 风格与要求
5. **文件生成** → 调用对应 Skill 产出 .docx / .xlsx / .pdf
6. **透明报告** → 在 claude_output.md 中汇总状态、来源、完成度
7. **最终校验** → 确保无幻觉、无违背 Prompt、无未声明假设

## 主要模块

### 需求层增强
- Prompt 关键词扫描与交付物清单自动生成
- 输出格式自动识别并调用对应 Skill
- 强制参照参考文件风格（如 bullet style、官方图标）

### 感知层增强
- 多路径网络数据获取（WebFetch → curl+代理 → 手动）
- 参考文件存在性、可读性、内容提取验证

### 推理层增强
- 数据来源强制标注
- 幻觉高危场景识别与预防
- 约束条件复述与歧义澄清

### 行动层增强
- 能力状态仪表盘报告
- 多级降级策略（推荐用户操作 → 提供脚本 → 部分完成）
- 部分完成时明确标注未完成项及原因

### 验证层增强
- 数据血缘表
- 假设声明清单
- 自我验证检查表

### 集成层增强
- 标准化调用 docx、xlsx、pdf 生成 Skill
- 生成真实可下载文件并在 claude_output.md 中引用

## 输出规范（评分系统兼容）

所有任务必须产出以下结构：
runs/task_X/
├── claude_output.md                  # 评分系统主读取文件（必选）
├── Proposed_Architecture_Summary.docx   # Prompt 要求的实际文件
├── Architecture_Diagram.pdf
├── POC_Guide.xlsx
└── auxiliary/                        # 辅助数据、HTML 中间件等
text`claude_output.md` 必须包含：
- Executive Summary
- Capability Status
- Deliverables Generated 表格
- Data Sources & Traceability
- Assumptions
- Completion Status

## 使用场景示例

- GCP 云迁移架构设计（生成 .docx 总结 + .pdf 架构图）
- 审计抽样分析（读取参考 Excel + 生成报告）
- S&P 500 公司财务指标分析（网络获取数据 + Excel 输出）
- 能源策略报告（多源数据整合 + Word/PDF 交付）
